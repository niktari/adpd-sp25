Description

As we learned last week, HTML and CSS is like a printed picture book that gets animated once JavaScript gets introduced into the fold. This week, we’ll talk more about how our static sites come to life by learning about the DOM, or the Document Object Model. We’ll then learn about hooking up HTML inputs to JavaScript so we can get our projects to start functioning.

  • Generative Tool, Part 3: Coded Draft

    Create a new subfolder called project-2 within your username.github.io folder. Create a new index.html, styles.css, and script.js file within that subfolder to work in. This week, you will be developing a coded draft of your website. By now, you should have a clear idea of the types of inputs that will drive your outputs. Don’t worry if you can’t get everything to work.

    If you are running into technical or code issues, this is the time to simplify the idea to make it more attainable for yourself. We recommend you start with smaller bits of your tool and work your way up. For instance, if one aspect of your tool is to change the background color based on user input, then start with that. We will learn how to use inputs next week, but for now, see if you can change your content or styles using JavaScript.

    Consider what aspects of your tool are interactive (driven by user input) and generative (infinite outputs based on a fixed set of parameters).
