  • Tool, Part 3: Coded Draft

    Create a new subfolder called project-3 within your username.github.io folder. Create a new index.html, styles.css, and script.js file within that subfolder to work in. Make sure you link your HTML file to your script.js by using the <script> tag. Reference this slide from last week if you need a refresher.

    This week, you will be developing a coded draft of your website. Choose one of the three directions and start to bring it into code. Focus on only getting the core functionality of the tool down in this round—no need to worry about the design/styling in this stage. You should have a clear idea of the types of inputs that will drive your outputs. Don’t worry if you can’t get everything to work.

    If you are running into technical or code issues, this is the time to simplify the idea to make it more attainable for yourself. I recommend you start with smaller bits of your tool and work your way up. For instance, if one aspect of your tool is to change the background color based on user input, then start with that.

    Consider generativity: how do the inputs of your tool serve as a fixed set of parameters that can generate a multitude (or infinite) amount of outputs?

    This assignment is worth 4 out of 25 points of Project 2: Generative Tool. To get full credit, you must hit all of the requirements below.
