Description

Welcome back from Spring Break! This week, we’ll continue our foray into JavaScript by learning about conditional logic to control the flow of our code and develop conditional-based designs. We’ll also finally dive into inputs: remember those button and checkbox HTML elements that you probably played around with in Week 1? We’ll learn how to hook those up to JavaScript to get those elements to actually work.

Assignments

  • Generative Tool, Part 4: Coded Prototype

    Continue to develop your coded draft into a viable prototype. Start to hook up your inputs to JavaScript. By now, you should have a clear idea of the types of inputs that will drive your outputs.

    If you are running into technical or code issues, this is the time to simplify the idea to make it more attainable for yourself. We recommend you start with smaller bits of your tool and work your way up. For instance, if one aspect of your tool is to change the background color based on user input, then start with that. Prioritize nailing down the main functionality of your tool in this round.

    Consider what aspects of your tool are interactive (driven by user input) and generative (infinite outputs based on a fixed set of parameters).
