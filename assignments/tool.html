Due April 21

In this project, you’ll explore the boundaries of what a tool can be when made on the web. Your goal is to create a website that generates infinite iterations based on a set of rules, driven by user input. This could take the form of a practical tool, like a type tester or image generator, or something more abstract and experimental.

It’s important to test your tool throughout the design process—actively engage with your friends, peers, and other users to gather feedback and improve the experience.

The requirements of this project are below:

  1. The user should be able to provide an input (e.g., text, selection, button click) and receive a unique output each time.

  2. The tool should be developed using HTML, CSS, and JavaScript.

  3. Include text that explains the purpose of the tool, how it works, and its intended use.

  4. Bonus: Make the tool responsive and work across different screen sizes.

  • Part 1: Concept & Sketch

    Using Figma, sketch three distinct ideas for a website that utilizes an input and an output. Determine what the input and output would be. For instance, when the user types something, will text appear? Consider who or what this tool is for. Do you want to make something that is useful, semi-useful or not useful at all?

  • Part 2: Coded Draft

    Choose one of the three directions and start to bring it into code. Focus on only getting the core functionality of the tool down in this round—no need to worry about the design/styling in this stage. If you are running into major technical issues, this is the time to simplify the idea to make it more achievable within the project scope.

  • Part 3: Refinement & Testing

    Refine your work based on class feedback. Finalize the technical functionality. Start to style up the tool, ensuring it aligns with its purpose and audience. Test the tool with at least three different people. Your testers should give you insight into the usability of your tool. Adjust your design and functionality based on the feedback.

  • Part 4: Finalize

    Based on class feedback, finalize your tool. Put together a 3-minute presentation for Project 3 that walks us through your design process.
